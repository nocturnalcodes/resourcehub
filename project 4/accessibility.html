Web Accessibility

Summary

Web accessibility focuses on creating a website that can be accessed and used by all user groups regardless of disabilities. Websites should be able to be easily navigated and understood even if users have cognitive, learning, or motor disabilities. Creating a website that is accessible is a great web development practice that improves overall user experience.

Techniques

  • Use alt text on images so they can be interpreted by screen readers.
  • Implement readable fonts and proper heading structure (H1–H6).
  • Provide distinguishable contrast between text and background.
  • Support full keyboard navigation.
  • Add captions for audio and video content.

Insights Gained in the Class

Although I understood how important website accessibility is, I did not realize how many techniques could be implemented to make a website more accessible. Furthermore, Web Content Accessibility Guidelines (WCAG) are the driving force behind ensuring web content is easily comprehendable and remains accessibile to those with disabilities. I also learned how to apply the four principles of web accessibility (Perceivable, Operable, Understandable, and Robust (POUR)) to analyze websites for the compliance with the WCAG. During this class, I learned about assistive technologies that help users with disabilities navigate the web, which gave me a deeper appreciation for inclusive digital design.

Resources for Website Accessibility

  • WebAIM Contrast Checker – Verifies text contrast meets WCAG requirements.
  • Accessible Web – A free website accessibility checker.
  • Lighthouse Accessibility Audit – Audits accessibility directly in Chrome DevTools.
