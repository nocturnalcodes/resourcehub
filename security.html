Web Application Security

Summary

Web application security involves protecting websites and online systems from malicious attacks, unauthorized access, and data exposure. Implementing effective security techniques protects user data, maintains system functionality, and ensures a safe digital experience for users.

Techniques

  • Validate user input to prevent SQL injection.
  • Enforce strong user authentication and password hashing.
  • Set proper permissions to prevent unauthorized data access.
  • Use Content Security Policy (CSP) to block unauthorized scripts.
  • Keep frameworks and libraries updated to avoid known vulnerabilities.

Insights Gained in the Class

In this class, I learned about the OWASP Web Application Security Project, a framework that identifies and categorizes common web security risks. I gained insight into how weak authentication, SQL injection, broken access control, and other vulnerabilities can compromise web applications. Understanding these risks emphasized the importance of proactive security measures to protect users and maintain trust in digital systems.

Resources for Web Application Security

  • OWASP ZAP (Zed Attack Proxy) – Security scanning tool used to detect vulnerabilities like XSS, injection flaws, and insecure configurations.
  • NPMScan – Quick check of npm/Node.js dependencies for vulnerabilities and supply‑chain risks.
  • Acunetix Vulnerability Scanner – Web-focused scanner that tests for a wide range of application vulnerabilities.
